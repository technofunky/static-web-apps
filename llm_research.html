🧠 LLM Post-Training & Agentic RL

Visual summaries + decision flowcharts | Based on two comprehensive surveys (2025–2026)

📌 Post-Training Decision Flowchart (from survey)
📌 TTC = Test-Time Scaling. GRPO removes critic, DPO removes reward model.

🎯 1. Introduction

Pages 1-2

Pre-trained LLMs excel at next-token prediction but hallucinate and misalign. Post-training (fine‑tuning, RL, test‑time scaling) fixes reasoning gaps.

🧩 2. Background (MDP & RL)

Pages 3-4

Text generation = MDP: state = context, action = next token. Policy gradient with advantage A(s,a)=Q−V.

⚙️ 3. Reward Modeling

Pages 6-7

Bradley‑Terry / Plackett‑Luce from human preferences. Outcome vs Process rewards. Iterative RL to avoid reward hacking.

🔄 4. PPO / DPO / GRPO

Pages 8-11

PPO uses critic + clipping; DPO eliminates reward model via log‑ratio loss; GRPO uses group statistics as baseline (DeepSeek).

⏱️ 5. Test‑Time Scaling

Pages 14-19

CoT, Self‑Consistency, Tree‑of‑Thoughts, Compute‑Optimal Scaling (4× efficiency). Smaller model + TTC can outperform 14× larger model.

📊 6. Benchmarks

Pages 19-20

Reasoning (MATH, GSM8K), RL alignment (HelpSteer, UltraFeedback), Multilingual, Dialogue, Search.

🧠 Agentic RL Paradigm Shift & Decision Map

🎭 2. Formal Shift (POMDP)

Pages 5-7

PBRFT = degenerate MDP (T=1). Agentic RL = POMDP with partial observations, action space A_text ∪ A_action, step‑wise rewards.

🧰 3. Capabilities

Pages 9-20

RL for planning (MCTS / internal policy), tool use (ToolRL, AutoTIR), memory (MemAgent, Memory‑R1), self‑improvement (Reflexion, R‑Zero).

📱 4. Applications

Pages 20-38

Web search, code generation (SWE-bench), mathematics (Lean/Isabelle), GUI navigation, embodied agents (Voyager), multi‑agent (MAGRPO).

🏞️ 5. Environments

Pages 38-42

WebArena, OSWorld, AndroidWorld, Debug-Gym, Crafter, SMAC. Frameworks: OpenRLHF, SkyRL, AgentFly, AWorld.

🔐 6. Challenges

Pages 43-48

Security (reward hacking), hallucination (outcome‑only RL worsens it), sycophancy, scaling compute/environments, mechanistic debate (amplifier vs new‑knowledge).

🔁 Integrated Pipeline: Post‑Training → Agentic RL
📌 Key synergy Post‑training (SFT+RL) prepares the model; Agentic RL deploys it in dynamic worlds with tool use, memory, and planning — together forming the full stack of modern reasoning agents (DeepSeek‑R1, OpenAI o3, Kimi K2).

🔄 Post‑Training → Agentic RL

Post‑training (fine‑tuning + DPO/GRPO) makes LLMs reliable and aligned. Agentic RL extends to multi‑turn POMDP environments with tool use, memory, and search.

⚡ Common Algorithms

GRPO and PPO are central to both paradigms: GRPO for efficiency (DeepSeek‑R1), PPO for stability. DPO simplifies preference alignment.

🧪 Real‑world Agents

OpenAI Deep Research, Kimi K2, WebAgent‑R1, Voyager, SWE agents use RL‑optimized planning + tool invocation. Scaling laws show RL fine‑tuning expands reasoning boundaries.

🚀 Future Trajectory

Hybrid systems: SFT for grounding, RL (GRPO) for reasoning, test‑time scaling for hard queries, and agentic frameworks for long‑horizon tasks.
